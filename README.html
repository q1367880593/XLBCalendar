XLBCalendar

一个把 LPL 赛程自动整理成日历文件的工具

⚡

工作流程

📡 腾讯 LPL 接口
→
📦 原始 JSON
→
🔗 合并解析
→
📅 ICS 日历
→
📲 导入应用
✨

核心功能

📡

自动拉取

从腾讯 LPL 接口获取最新比赛数据，保存原始 JSON 到 raw/ 目录

🔀

数据合并

统一读取多源 JSON，按比赛日期排序后合并为完整数据集

📅

ICS 日历

生成标准 .ics 文件，可直接导入系统日历、Apple 日历、Google 日历

🎯

双日历输出

LPL 全量赛程 + BLG 专属赛程，满足不同观赛需求

📊

赛段统计

按赛段自动统计队伍胜负关系，生成可视化拓扑结构

🔄

一键部署

auto_lpl.sh 脚本集成拉取、生成、提交、推送全流程

📁

项目文件

PY
LPL_T.py 拉取 LPL 接口数据 → 保存原始 JSON → 解析比赛结果 → 按赛段统计胜负 → 控制台输出日志
PY
convert_to_ics.py 读取 raw/ 下 JSON → 合并排序 → 转换为 .ics 日历 → 生成全量 & BLG 双日历
SH
auto_lpl.sh 一键执行数据拉取、日历生成、git 提交和推送
GI
.gitignore 忽略缓存文件和中间产物
▶️

运行方式

确保本机有 Python 环境，可使用 python3 命令。

一键执行 
bash auto_lpl.sh
分步执行 
# 第一步：拉取数据
python3 LPL_T.py

# 第二步：生成日历
python3 convert_to_ics.py

核心逻辑

只统计 MatchStatus == "3" 的比赛，即已结束的比赛
合并 raw/ 下 JSON 后按 MatchDate 排序，逐场写入日历事件
UID 基于 bMatchId 生成，方便日历应用识别与更新同一事件
SEQUENCE 用于区分未开始和已结束比赛状态
日历标题前缀根据 GameName / GameTypeName 自动映射（LPL、FST、MSI）
队伍名缺失时从 bMatchName 中按 vs 自动解析
拓扑结构输出同样携带赛事前缀，便于区分来源
⚠️

注意事项

💡
  • 接口字段名变更时，脚本可能需要同步调整
  • 每次运行前 raw/ 会被清空，仅保留本次抓取的数据
  • raw/ 中的 JSON 为中间产物，默认不提交到仓库
  • json.json 为合并后的中间产物，默认不提交到仓库
